Hermes Dashboard

Planner → Generator → Evaluator

Navegue pelo Hermes Dashboard sem se perder em uma página corrida.

Este guia transforma o tutorial em uma aplicação de navegação lateral: escolha um tópico, veja apenas o conteúdo necessário, abra detalhes sob demanda e valide tudo com um checklist de QA.

Fluxo operacional do manual

Planner

Escolhe o caminho correto: local, VPS ou túnel SSH.

Generator

Executa comandos copiáveis com contexto e ordem clara.

Evaluator

Valida com curl, lsof, logs e checklist antes de confiar.

Acesso local

Suba o dashboard em localhost.

Use quando estiver dentro da própria máquina ou container. É o modo seguro porque não expõe o dashboard na rede.

Fluxo local

Iniciar dashboardBind local seguro
hermes dashboard --host 127.0.0.1 --port 9119 --no-open
Validar APIDeve retornar JSON
curl http://127.0.0.1:9119/api/status

Hostinger · Docker · Mac

Use SSH Tunnel para acessar o dashboard remoto.

O container escuta em 0.0.0.0 para o Docker mapping funcionar, mas o acesso externo real passa pelo túnel SSH.

Fluxo remoto seguro

1

docker-compose.yaml

hPanel → VPS → Gerenciador Docker → Editor .yaml.

ports
ports:
  - "4860"
  - "9119:9119"
2

Container

Entre na VPS e no container Hermes.

Acesso
ssh root@76.13.224.9
docker exec -it hermes-agent-vt3l-hermes-agent-1 bash
3

Dashboard

Reinicie evitando processo duplicado.

Reiniciar
pkill -f "hermes dashboard" 2>/dev/null
sleep 2
nohup hermes dashboard --host 0.0.0.0 --port 9119 --insecure --no-open > /opt/data/dashboard.log 2>&1 &
4

Mac

Crie o túnel e abra no navegador local.

Túnel SSH
ssh -f -N -o ServerAliveInterval=60 -L 9119:127.0.0.1:9119 root@76.13.224.9
