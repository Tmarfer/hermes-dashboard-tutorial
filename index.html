Passo a passo

Como iniciar o servidor local

  1. Abra o terminalUse a máquina onde o Hermes Agent está instalado.
  2. Suba o dashboardExecute o comando abaixo para escutar somente em localhost.
  3. Abra o navegadorAcesse http://127.0.0.1:9119.
  4. Confirme a APISe /api/status responder, o dashboard está ativo.

Comando recomendado

hermes dashboard --host 127.0.0.1 --port 9119 --no-open

Teste rápido

curl http://127.0.0.1:9119/api/status

VPS ou servidor remoto

Como acessar de fora sem expor o painel

Se o Hermes roda em uma VPS, mantenha o dashboard em 127.0.0.1 e use um túnel SSH. Assim o painel continua protegido.

ssh -L 9119:127.0.0.1:9119 usuario@IP_DA_VPS

Depois de conectar o túnel, abra no seu navegador local: http://127.0.0.1:9119

Segurança

Não exponha suas chaves por acidente

O dashboard consegue ler e alterar o arquivo .env, onde ficam API keys e tokens. Por isso, o modo seguro é rodar em 127.0.0.1.

Seguro

--host 127.0.0.1

Acesso local ou via túnel SSH.

Evite sem proteção

--host 0.0.0.0 --insecure

Só use com VPN, firewall, autenticação/reverse proxy e consciência do risco.

FAQ

Dúvidas comuns

O navegador não abre. O que faço?

Acesse manualmente http://127.0.0.1:9119. Se estiver em VPS, crie o túnel SSH.

Como sei se está rodando?

Teste com curl http://127.0.0.1:9119/api/status ou verifique a porta com ss -ltnp.

Posso compartilhar o dashboard na internet?

Não diretamente. Ele não deve ser exposto sem autenticação e controle de rede, pois permite gerenciar secrets.

Alterações no config.yaml valem na hora?

Algumas mudanças exigem nova sessão ou restart do gateway/processo para efeito completo.
