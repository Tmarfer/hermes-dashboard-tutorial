Harness design aplicado ao tutorial

O manual foi reorganizado como um pequeno harness de operação.

Inspirado na engenharia da Anthropic: quem planeja, quem executa e quem avalia não devem ser a mesma função mental. Isso reduz erro, pressa e autoavaliação generosa demais.

01

Planner

Define o caminho: local, Docker, VPS ou túnel SSH. Escolhe o roteiro certo antes de rodar comandos.

02

Generator

Executa comandos com blocos copiáveis e passos numerados. Evita improviso e mantém rastreabilidade.

03

Evaluator / QA

Valida com curl, lsof, logs e checklist final. Só considera pronto quando o JSON e o navegador respondem.

Acesso local

Suba o dashboard sem expor suas chaves.

Use este modo quando você está dentro da própria máquina/container e quer validar o painel com segurança.

1Abra o terminal na máquina onde o Hermes Agent está instalado.
2Inicie o dashboard bindado em localhost.
3Teste a API antes de abrir o navegador.
IniciarSeguro/local
hermes dashboard --host 127.0.0.1 --port 9119 --no-open
ValidarDeve retornar JSON
curl http://127.0.0.1:9119/api/status

Hostinger · Docker · SSH Tunnel

Acesse o dashboard remoto sem abrir a porta 9119 ao público.

O Docker precisa encaminhar a porta até a VPS, mas o acesso de fora continua protegido pela porta 22 do SSH.

Docker mapping

9119:9119 sem 127.0.0.1: no host.

Container bind

--host 0.0.0.0 --insecure dentro do container.

Mac access

ssh -L 9119:127.0.0.1:9119 até a VPS.

hPanel

1. Ajustar docker-compose.yaml

Hostinger → VPS → Gerenciador Docker → container hermes-agent → Editor .yaml.

ports:
  - "4860"
  - "9119:9119"

Remover 127.0.0.1: do lado do host permite o Docker proxy encaminhar a conexão.

VPS

2. Entrar no container

ssh root@76.13.224.9
docker exec -it hermes-agent-vt3l-hermes-agent-1 bash
Container

3. Reiniciar dashboard

pkill -f "hermes dashboard" 2>/dev/null
sleep 2
nohup hermes dashboard --host 0.0.0.0 --port 9119 --insecure --no-open > /opt/data/dashboard.log 2>&1 &
Teste

4. Validar container e VPS

curl http://127.0.0.1:9119/api/status
exit
curl http://127.0.0.1:9119/api/status
Mac

5. Criar e validar túnel SSH

ssh -f -N -o ServerAliveInterval=60 -L 9119:127.0.0.1:9119 root@76.13.224.9
lsof -i :9119
curl http://127.0.0.1:9119/api/status

Depois abra http://127.0.0.1:9119 no navegador.
