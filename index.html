O que é

O Hermes Dashboard em uma frase

É um painel web local do Hermes Agent para operar e auditar o ambiente sem depender só de comandos no terminal.

Veja versão, gateway, sessões ativas e estado geral da instalação.

Operação do agente

Consulte logs, sessões, analytics, cron jobs, skills e toolsets.

Passo a passo

Como iniciar o servidor local

  1. Abra o terminalUse a máquina onde o Hermes Agent está instalado.
  2. Suba o dashboardExecute o comando abaixo para escutar somente em localhost.
  3. Abra o navegadorAcesse http://127.0.0.1:9119.
  4. Confirme a APISe /api/status responder, o dashboard está ativo.

Comando recomendado

hermes dashboard --host 127.0.0.1 --port 9119 --no-open

Teste rápido

curl http://127.0.0.1:9119/api/status

VPS ou servidor remoto

Como acessar de fora sem expor o painel

Se o Hermes roda em uma VPS, mantenha o dashboard em 127.0.0.1 e use um túnel SSH. Assim o painel continua protegido.

ssh -L 9119:127.0.0.1:9119 usuario@IP_DA_VPS

Depois de conectar o túnel, abra no seu navegador local: http://127.0.0.1:9119

Hostinger + Docker + SSH

Aba SSH Tunnel: acesso seguro ao dashboard sem expor a porta publicamente

Este roteiro cobre o cenário em que o Hermes Agent roda dentro de um container Docker na Hostinger e o painel precisa ser acessado pelo Mac via túnel SSH.

Objetivo

Acessar o Hermes Dashboard na porta 9119 usando SSH, sem abrir a porta 9119 no firewall público.

Estratégia

Docker publica 9119 no host da VPS, mas o acesso externo real acontece pelo túnel SSH na porta 22.

Segurança

O dashboard usa --insecure apenas dentro do caminho container → VPS → túnel SSH.

1. docker-compose.yaml no hPanel

No hPanel da Hostinger, vá em VPS → Gerenciador Docker → container hermes-agent → Editor .yaml. Na seção ports:, use:

ports:
  - "4860"
  - "9119:9119"

Importante: não use 127.0.0.1:9119:9119 no lado do host. Remover 127.0.0.1: permite que o Docker proxy encaminhe a conexão para o container.

01

Acessar hPanel

Entre em https://hpanel.hostinger.com, selecione a VPS e abra o Gerenciador Docker.

02

Editar YAML

Selecione o container Hermes Agent, abra o Editor .yaml e ajuste a seção ports.

03

Implantar

Clique em Implantar e aguarde o container voltar para Running, normalmente cerca de 30 segundos.

2. Entrar na VPS e no container

ssh root@76.13.224.9
docker exec -it hermes-agent-vt3l-hermes-agent-1 bash

3. Reiniciar o dashboard dentro do container

Mate processos antigos para evitar o erro address already in use e suba o dashboard aceitando conexão do Docker mapping:

pkill -f "hermes dashboard" 2>/dev/null
sleep 2
nohup hermes dashboard --host 0.0.0.0 --port 9119 --insecure --no-open > /opt/data/dashboard.log 2>&1 &

4. Validar dentro do container

sleep 5
curl http://127.0.0.1:9119/api/status
exit

O resultado esperado é um JSON. Se falhar, veja /opt/data/dashboard.log.

5. Validar na VPS, fora do container

curl http://127.0.0.1:9119/api/status

Se retornar JSON, o Docker mapping 9119:9119 está funcionando.

6. Criar túnel SSH no Mac

ssh -f -N -o ServerAliveInterval=60 -L 9119:127.0.0.1:9119 root@76.13.224.9

Digite a senha quando solicitado. Se nada aparecer depois, é normal: o processo fica em background.

7. Validar túnel no Mac e abrir o navegador

lsof -i :9119
curl http://127.0.0.1:9119/api/status

Depois abra: http://127.0.0.1:9119. O dashboard deve carregar.

Nota de segurança

O dashboard roda com --insecure porque precisa escutar em 0.0.0.0 dentro do container. A proteção vem do fato de que o acesso externo é feito por túnel SSH, não por porta pública.

Persistência

O comando com nohup ... & não sobrevive necessariamente a reboot/recriação do container. Para produção, considere um segundo serviço hermes-dashboard no docker-compose.

Segurança

Não exponha suas chaves por acidente

O dashboard consegue ler e alterar o arquivo .env, onde ficam API keys e tokens. Por isso, o modo seguro é rodar em 127.0.0.1.

Seguro

--host 127.0.0.1

Acesso local ou via túnel SSH.

Evite sem proteção

--host 0.0.0.0 --insecure

Só use com VPN, firewall, autenticação/reverse proxy e consciência do risco.

FAQ

Dúvidas comuns

O navegador não abre. O que faço?

Acesse manualmente http://127.0.0.1:9119. Se estiver em VPS, crie o túnel SSH.

Como sei se está rodando?

Teste com curl http://127.0.0.1:9119/api/status ou verifique a porta com ss -ltnp.

Posso compartilhar o dashboard na internet?

Não diretamente. Ele não deve ser exposto sem autenticação e controle de rede, pois permite gerenciar secrets.

Alterações no config.yaml valem na hora?

Algumas mudanças exigem nova sessão ou restart do gateway/processo para efeito completo.
